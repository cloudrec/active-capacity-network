# Node Roles — Active Capacity Network (Preview)

All roles are **private-preview**. None earn rewards, mine, or move value.

| Role | Who | What it does (preview) | Hardware | Rewards |
|------|-----|------------------------|----------|---------|
| **Light Node** | Everyone (recommended) | Read-only: shows live network summary, validators, epochs, sync status. | Any Win10/11 PC, Python 3.9+. | None |
| **Capacity Node** | Contributors evaluating the capacity role | Preview of the capacity-contribution view. Read-only in this client. | Modest always-on PC. | None |
| **Validator Node** | Vetted operators | **Documentation + application only.** Real validation is admission-gated on the network; this client cannot self-approve or validate. | Reliable uptime, reviewed by admins. | None in preview |

## How to progress
1. Start as a **Light Node** (`START_LIGHT_NODE.bat`).
2. Read the role docs and the consensus overview on `capacity.469diamond.com`.
3. To be considered as a validator operator, submit the **validator-apply** form on the site
   (`/#/validator-apply`). Admission is manual and reviewed — never automatic from this package.

## Honest limits
- This is a preview client over **public read-only APIs**. It does not run consensus locally.
- "Capacity" and "Validator" modes here are **previews/onboarding**, not live network duties.
- No role generates income. Any future reward model would be announced separately and is not implied.
